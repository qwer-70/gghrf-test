# Этап 08. Итоговое консольное приложение

## Задача финального этапа

На финальном этапе нужно собрать слои проекта в одну работающую программу.

Интерфейс может быть простым, но он должен позволять пройти основной путь покупателя: каталог, поиск, корзина, скидка, заказ.

К этому моменту код уже должен быть распределен по файлам этапов. Финальный этап не создает копии классов, а импортирует готовые модели, репозитории и сервисы из предыдущих `tasks.py`.

## Интерфейсный слой

Интерфейсный слой отвечает за:

- показ меню;
- чтение ввода пользователя;
- вывод результатов;
- преобразование строкового ввода в нужные значения;
- показ ошибок понятным текстом.

Он не должен хранить бизнес-правила. Например, проверка остатков должна оставаться в сервисе, а не в обработчике пункта меню.

## Сборка приложения

В начале программы обычно создаются:

- подключение к PostgreSQL;
- репозитории;
- сервисы;
- стартовые данные;
- текущая корзина;
- объект интерфейса.

После этого запускается главный цикл меню.

Стартовые данные лучше добавлять так, чтобы повторный запуск программы не создавал дубликаты. Например, перед добавлением можно проверять, есть ли уже нужная категория, товар или промокод.

## Главное меню

Для учебного проекта достаточно такого меню:

- показать каталог;
- найти товар;
- добавить товар в корзину;
- изменить корзину;
- посмотреть корзину;
- применить промокод;
- оформить заказ;
- посмотреть историю заказов покупателя;
- выйти.

Меню можно расширять, но сначала лучше добиться стабильной работы основного сценария.

## Обработка ошибок

Пользователь может ввести неверное число, неизвестный идентификатор, неподходящий размер или несуществующий промокод.

Программа не должна завершаться аварийно. Интерфейс должен показать сообщение и вернуть пользователя в меню.

## Пример из другой области

Представим консольное приложение для списка задач. Интерфейс показывает меню и читает ввод, а работа с задачами находится в отдельном сервисе.

```python
class TaskConsoleApp:
    def __init__(self, task_service):
        self.task_service = task_service

    def show_menu(self):
        print("1. Показать задачи")
        print("2. Добавить задачу")
        print("0. Выйти")

    def run(self):
        while True:
            self.show_menu()
            command = input("Выберите действие: ")

            if command == "1":
                tasks = self.task_service.get_all_tasks()
                for task in tasks:
                    print(task.title)
            elif command == "2":
                title = input("Название задачи: ")
                self.task_service.create_task(title)
            elif command == "0":
                break
            else:
                print("Неизвестная команда")
```

Такой класс отвечает за диалог с пользователем. Правила создания, поиска и изменения задач остаются в сервисах.
