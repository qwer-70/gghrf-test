# PostgreSQL для проекта

## Что нужно установить

Для проекта нужна реальная PostgreSQL-база данных.

ORM использовать нельзя. Для подключения из Python нужен только драйвер:

```bash
pip install "psycopg[binary]"
```

Можно использовать `psycopg2`, если преподаватель выбрал его заранее, но в одном проекте лучше не смешивать разные драйверы.

## Параметры подключения

Не храните пароль от базы прямо в коде.

Удобный вариант - переменные окружения:

```bash
DB_HOST=localhost
DB_PORT=5432
DB_NAME=clothing_store
DB_USER=postgres
DB_PASSWORD=postgres
```

В Python их можно читать через стандартный модуль `os`.

## Создание базы

Базу можно создать через `psql`, pgAdmin или другой клиент PostgreSQL.

Пример команды:

```sql
CREATE DATABASE clothing_store;
```

## SQL-схема

Схема проекта находится в папке:

```text
02_postgresql_storage/schema.sql
```

Этот файл специально оставлен как каркас. Студенты должны самостоятельно описать таблицы, связи и ограничения.

Минимально в схеме должны появиться таблицы для категорий, товаров, остатков товаров по размерам, покупателей, адресов, промокодов, заказов и позиций заказов.

## Роль репозиториев

Репозитории должны:

- открывать курсор;
- выполнять SQL-запросы;
- передавать параметры отдельно от строки запроса;
- превращать строки из БД в объекты моделей;
- фиксировать изменения через `commit()`;
- откатывать изменения через `rollback()` при ошибках, если операция состоит из нескольких шагов.
