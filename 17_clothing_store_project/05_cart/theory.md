# Этап 05. Корзина

## Роль корзины

Корзина хранит выбранные покупателем товары до оформления заказа.

В корзине важна не только ссылка на товар, но и выбранный размер, количество и цена на момент добавления. Для учебного проекта можно считать итоговую сумму по текущей цене товара, но в позиции корзины все равно полезно хранить цену явно.

В этом проекте корзина может быть временной и жить в памяти текущего запуска программы. Постоянно сохранять нужно оформленные заказы.

## Позиция корзины

Позиция корзины обычно отвечает на вопросы:

- какой товар выбран;
- какой размер выбран;
- сколько единиц добавлено;
- сколько стоит эта позиция.

Корзина отвечает за набор таких позиций и общий итог.

## Проверка остатков

При добавлении товара нужно проверить:

- товар активен;
- выбранный размер существует;
- количество больше нуля;
- на складе достаточно товара именно выбранного размера.

Списание со склада лучше делать не при добавлении в корзину, а при оформлении заказа. Пока товар лежит в корзине, покупатель еще не купил его.

## Где должна жить логика

Корзина может считать сумму и управлять позициями. Проверку доступности товара можно делать через сервис каталога или отдельный сервис корзины.

Главная мысль: пользовательский ввод не должен быть внутри корзины.

## Пример из другой области

Представим список выбранных билетов в кинотеатре. Пользователь выбирает сеанс, места и количество билетов. До оплаты это еще не покупка, а только временный набор выбранных позиций.

```python
class TicketSelection:
    def __init__(self, movie_title, seat, price):
        self.movie_title = movie_title
        self.seat = seat
        self.price = price


class TicketCart:
    def __init__(self):
        self._items = []

    def add_ticket(self, ticket):
        self._items.append(ticket)

    def total_price(self):
        total = 0

        for ticket in self._items:
            total += ticket.price

        return total

    def clear(self):
        self._items.clear()
```

Такой объект отвечает за выбранные позиции и сумму, но не занимается оплатой, печатью билетов или сохранением данных.
