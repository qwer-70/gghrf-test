# Этап 04. Сервис каталога

## Что такое сервис

Сервис содержит бизнес-логику приложения. Он использует модели и репозитории, но сам не занимается вводом с клавиатуры и печатью меню.

Если репозиторий отвечает на вопрос "где хранятся товары", то сервис каталога отвечает на вопросы:

- какие товары можно показывать покупателю;
- как искать товар;
- как фильтровать каталог;
- доступен ли товар для покупки;
- как сортировать выдачу.

## Почему поиск не стоит писать в интерфейсе

Интерфейс может быть консольным, графическим или веб-интерфейсом. Логика поиска от этого не меняется.

Если поиск живет в сервисе, его можно использовать из любого интерфейса.

## Пример из другой области

Представим сервис расписания занятий. Репозиторий хранит занятия, а сервис умеет выбирать только те занятия, которые нужны пользователю.

```python
class LessonScheduleService:
    def __init__(self, lesson_repository):
        self.lesson_repository = lesson_repository

    def find_by_teacher(self, teacher_name):
        lessons = self.lesson_repository.get_all()
        result = []

        for lesson in lessons:
            if lesson.teacher_name.lower() == teacher_name.lower():
                result.append(lesson)

        return result

    def get_active_lessons(self):
        lessons = self.lesson_repository.get_all()
        return [lesson for lesson in lessons if lesson.is_active]
```

Сервис не спрашивает ввод у пользователя и не печатает меню. Он получает данные из репозитория, применяет правила и возвращает результат.

## Фильтрация и сортировка

Для каталога часто нужны фильтры:

- по категории;
- по цвету;
- по размеру;
- по диапазону цены;
- по наличию на складе.

Для одежды фильтр по размеру должен проверять остаток конкретного размера, а не просто наличие размера в списке.

Сортировка может быть по цене, названию или остатку. На этом этапе достаточно нескольких вариантов.

## Активные и неактивные товары

Товар может быть временно скрыт из каталога. Например, его сняли с продажи, но удалять историю не хочется.

Сервис каталога должен учитывать это правило и не показывать неактивные товары в обычной выдаче.
