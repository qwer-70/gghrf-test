# Этап 07. Пользователи и скидки

## Покупатель и доставка

В магазине покупатель обычно имеет контактные данные и адрес доставки. Эти данные лучше хранить отдельно от заказа, а при оформлении заказа фиксировать нужную информацию.

Адрес можно описать отдельной моделью, чтобы не превращать покупателя в набор несвязанных строк.

Для адресов нужна отдельная таблица. Она должна ссылаться на покупателя через внешний ключ.

## Промокод

Промокод - это отдельная сущность. У него могут быть:

- код;
- процент скидки;
- минимальная сумма заказа;
- признак активности;
- ограничение по количеству использований.

Даже если в учебном проекте нужны не все поля, полезно держать промокод отдельно от корзины и заказа.

Для промокодов нужна отдельная таблица. Код промокода лучше сделать уникальным на уровне БД.

## Сервис скидок

Сервис скидок проверяет, можно ли применить промокод, и считает новую сумму.

Корзина не должна сама искать промокоды. Заказ не должен сам решать, активна ли скидка. Так проект остается разделенным по обязанностям.

## Деньги и округление

В учебном проекте можно использовать `int` для цен в рублях. Это проще, чем работать с копейками и округлением.

Если используется `float`, нужно помнить, что вычисления с дробными числами могут давать неточные результаты.

## Пример из другой области

Представим сервис скидок для онлайн-курсов. Промокод применяется только к подходящей сумме и только если он активен.

```python
class CoursePromoCode:
    def __init__(self, code, percent, min_total, is_active=True):
        self.code = code
        self.percent = percent
        self.min_total = min_total
        self.is_active = is_active


class CourseDiscountService:
    def calculate_total(self, total, promo_code):
        if promo_code is None:
            return total

        if not promo_code.is_active:
            raise ValueError("Промокод неактивен")

        if total < promo_code.min_total:
            raise ValueError("Сумма меньше минимальной")

        discount = total * promo_code.percent // 100
        return total - discount
```

Сервис скидок не хранит курсы и не оформляет покупку. Он отвечает только за проверку промокода и расчет суммы.
